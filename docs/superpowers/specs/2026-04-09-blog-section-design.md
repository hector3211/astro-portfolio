# Blog Section Design

**Date:** 2026-04-09
**Status:** Approved

## Overview

Add a `/blog` section to the existing Astro portfolio site. Blog posts are authored as local markdown files. The listing page shows title + date (minimal). Individual post pages render markdown with syntax highlighting and full SEO metadata.

## Storage

- Posts live in `src/content/blog/` as `.md` files.
- Frontmatter schema (Zod-enforced):
  - `title` (string, required)
  - `date` (date, required)
  - `description` (string, required â€” used for meta tags and SEO)

## SEO

Every blog post page includes:
- `<title>{title} | Hector Oropesa</title>`
- `<meta name="description">` from frontmatter `description`
- Open Graph tags: `og:title`, `og:description`, `og:url`, `og:type`
- Twitter Card tags: `twitter:card`, `twitter:title`, `twitter:description`
- `<link rel="canonical">` pointing to the canonical URL
- JSON-LD `BlogPosting` structured data (title, date, description, URL)
- Semantic HTML: post body in `<article>`, date in `<time datetime="...">`, proper heading hierarchy

The sitemap at `/sitemap.xml` is auto-generated by `@astrojs/sitemap` and covers all blog post URLs alongside the homepage.

- All posts sorted by date descending (newest first)
- Each post row: `title` (link to post) + `date` (formatted, e.g. `April 9, 2026`)
- Minimal layout consistent with the existing site style
